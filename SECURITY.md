# Security Policy

## Reporting a Vulnerability

Please report security issues privately to the project maintainers instead of opening a public issue.

- Primary contact: `security@pilma.local`
- Backup contact: `maintainers@pilma.local`

Include:

- affected version or commit
- reproduction steps
- impact assessment
- any suggested mitigation

## Response Targets

- We aim to acknowledge incoming reports within 14 days.
- We aim to ship or communicate a remediation plan for confirmed issues within 60 days.

## Supported Scope

This policy covers:

- the localhost companion service
- vault storage and token handling
- trace emission and logging behavior
- dependency and workflow security for this repository

## Safe Harbor

Good-faith research that avoids data exfiltration, service abuse, or privacy violations will be treated as responsible disclosure.
